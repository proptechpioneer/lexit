### Prerequisites
- Railway account (https://railway.app)
- GitHub repository with your LEXIT code
- Email provider setup (Gmail recommended)

### 1. Connect Repository to Railway

1. Visit https://railway.app
2. Click "Start a New Project"
3. Select "Deploy from GitHub repo"
4. Choose your LEXIT repository
5. Railway will automatically detect it's a Django project

### 2. Environment Variables Configuration

In Railway dashboard, go to Variables tab and add:

```bash
# Core Django Settings
ENVIRONMENT=production
SECRET_KEY=your-super-secret-key-here
DJANGO_SETTINGS_MODULE=lexit.settings

# Database (Railway provides PostgreSQL automatically)
# DATABASE_URL will be auto-generated by Railway

# Email Configuration
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_HOST_USER=info@lexit.tech
EMAIL_HOST_PASSWORD=your-gmail-app-password
DEFAULT_FROM_EMAIL=info@lexit.tech
SERVER_EMAIL=info@lexit.tech

# Static Files (if using Cloudinary)
CLOUDINARY_CLOUD_NAME=your-cloudinary-name
CLOUDINARY_API_KEY=your-cloudinary-key
CLOUDINARY_API_SECRET=your-cloudinary-secret
```

### 3. Gmail Setup for info@lexit.tech

#### Option A: Gmail Workspace (Recommended for business)
1. Set up Google Workspace for lexit.tech domain
2. Create info@lexit.tech mailbox
3. Enable 2-factor authentication
4. Generate App Password:
   - Google Account > Security > 2-Step Verification > App passwords
   - Select "Mail" and generate password
   - Use this password in `EMAIL_HOST_PASSWORD`

#### Option B: Gmail Forwarding (Alternative)
1. Use personal Gmail account
2. Set up email forwarding to info@lexit.tech
3. Configure "Send mail as" info@lexit.tech in Gmail settings
4. Use personal Gmail credentials with custom from address

### 4. Domain Configuration

#### Custom Domain Setup:
1. In Railway dashboard, go to Settings > Domains
2. Add custom domain: `lexit.tech`
3. Add www subdomain: `www.lexit.tech`
4. Configure DNS records in your domain provider:
   ```
   Type: CNAME
   Name: @
   Value: [railway-provided-domain]
   
   Type: CNAME  
   Name: www
   Value: [railway-provided-domain]
   ```

### 5. Database Migration

Railway automatically provides PostgreSQL. After first deployment:

```bash
# Railway will run migrations automatically via Procfile
# Or manually trigger in Railway dashboard
railway run python manage.py migrate
railway run python manage.py collectstatic --noinput
```

### 6. Required Files for Railway

Ensure these files are in your repository:

#### `Procfile` (already exists):
```
web: python manage.py collectstatic --noinput && python manage.py migrate && gunicorn lexit.wsgi
```

#### `railway.json` (create if needed):
```json
{
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "python manage.py collectstatic --noinput && python manage.py migrate && gunicorn lexit.wsgi:application",
    "healthcheckPath": "/",
    "healthcheckTimeout": 300
  }
}
```

### 7. SSL Certificate

Railway automatically provides SSL certificates for custom domains.
Your site will be accessible at:
- https://lexit.tech
- https://www.lexit.tech

### 8. Testing Email Configuration

After deployment, test emails using the management command:

```bash
# Via Railway CLI
railway run python manage.py test_email your-test-email@gmail.com

# Or create a simple test view in your Django app
```
